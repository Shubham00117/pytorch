RNN / LSTM
What is RNN? A Recurrent Neural Network is designed for sequential data. Unlike feedforward networks that process inputs independently, RNNs maintain a memory of previous inputs using loops in their architecture. Well-suited for time series, speech recognition, and text generation.
Project Pipeline (5 Steps)
  • 1
    Load Dataset (CSV) — 100 unique Q&A pairs from Google Sheets. Columns: questionanswer
  • 2
    Text → Embeddings — Tokenize sentences → build vocabulary dict → convert words to numerical indices → embed into 50-dim vectors via nn.Embedding
  • 3
    Build RNN Model — SimpleRNN(nn.Module) with Embedding → RNN → Linear layers
  • 4
    Train — Adam optimizer, CrossEntropyLoss, 20 epochs, lr=0.001
  • 5
    Predict — Convert question → tensor → model → softmax → argmax → vocab word

Text Preprocessing

① Tokenize

def tokenize(text):
    text = text.lower()
    text = text.replace('?', '')
    text = text.replace(',', '')
    return text.split()

# Example output:
tokenize('What is the capital of France?')
# → ['what', 'is', 'the', 'capital', 'of', 'france']

② Build Vocabulary

# Initialize with unknown token
vocab = {'<UNK>': 0}

def build_vocab(row):
    tokens = tokenize(row['question']) \
           + tokenize(row['answer'])
    for token in tokens:
        if token not in vocab:
            vocab[token] = len(vocab)

df.apply(build_vocab, axis=1)
len(vocab)  # → 324

③ text_to_indices

def text_to_indices(text, vocab):
    indexed_text = []
    for token in tokenize(text):
        if token in vocab:
            indexed_text.append(vocab[token])
        else:
            indexed_text.append(vocab['<UNK>'])
    return indexed_text

# "What is the capital of France" → [1, 2, 3, 5, 6, 10]
Vocab size = 324 — This becomes the output dimension of the final Linear layer (one logit per word in vocab = one possible answer per vocab word).

PyTorch Dataset & DataLoader
class QADataset(Dataset):

    def __init__(self, df, vocab):
        self.df = df
        self.vocab = vocab

    def __len__(self):
        return self.df.shape[0]

    def __getitem__(self, index):
        q = text_to_indices(
            self.df.iloc[index]['question'],
            self.vocab)
        a = text_to_indices(
            self.df.iloc[index]['answer'],
            self.vocab)
        return (torch.tensor(q),
                torch.tensor(a))
# Instantiate
dataset = QADataset(df, vocab)

# DataLoader — batch_size=1
# (variable-length sequences)
dataloader = DataLoader(
    dataset,
    batch_size=1,
    shuffle=True
)
batch_size=1 is used because each question has a different number of tokens — no padding is applied in this simple version.
Model Architecture — SimpleRNN
class SimpleRNN(nn.Module):

    def __init__(self, vocab_size):
        super().__init__()
        # word index → 50-dim vector
        self.embedding = nn.Embedding(vocab_size, embedding_dim=50)
        # 50-dim input → 64-dim hidden state
        self.rnn = nn.RNN(50, 64, batch_first=True)
        # 64-dim → vocab_size logits (one per possible answer)
        self.fc = nn.Linear(64, vocab_size)

    def forward(self, question):
        embedded_question = self.embedding(question)       # [1, seq, 50]
        hidden, final = self.rnn(embedded_question)       # hidden: [1,seq,64], final: [1,1,64]
        output = self.fc(final)                            # [1, 1, 324]
        return output

Training Loop
# Hyperparameters
learning_rate = 0.001
epochs = 20

# Setup
model = SimpleRNN(len(vocab))
criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.Adam(
    model.parameters(),
    lr=learning_rate
)
for epoch in range(epochs):
    total_loss = 0
    for question, answer in dataloader:
        optimizer.zero_grad()          # clear grads
        output = model(question)        # forward
        loss = criterion(output, answer)# loss
        loss.backward()                # backprop
        optimizer.step()               # update
        total_loss += loss.item()
    print(f"Epoch: {epoch+1}, Loss: {total_loss:.4f}")

def predict_with_confidence(model, question_text, threshold=0.5):
    # 1. Convert question to indices
    question = text_to_indices(question_text, vocab)
    question_tensor = torch.tensor(question).unsqueeze(0)

    # 2. Forward pass
    output = model(question_tensor)              # [1,1,324]

    # 3. Convert logits → probabilities
    probs = torch.nn.functional.softmax(output, dim=1)

    # 4. Get most confident prediction
    confidence, predicted_index = torch.max(probs, dim=1)

    # 5. Threshold check
    if confidence.item() < threshold:
        return "I don't know"

    return list(vocab.keys())[predicted_index]

⚙️ Key Design Choices

  • threshold=0.5 — if model's max confidence < 0.5, return "I don't know"
  • softmax dim=1 — over vocab dimension
  • unsqueeze(0) — adds batch dimension
  • Answers are looked up by index in vocab keys list
Key Concepts & Gotchas

🔑 batch_first=True

Without this flag, RNN expects input as [seq, batch, features]. Setting batch_first=True makes it [batch, seq, features] — consistent with how DataLoader returns data.

🔑 Final vs All Hidden

y(x)[0] = all hidden states (shape: [B, seq, 64])
y(x)[1] = final hidden only (shape: [1, B, 64])
We use [1] for classification.

🔑 CrossEntropyLoss

Expects output: [batch, classes] and target: [batch] (class indices, not one-hot). The answer tensor from vocab index serves directly as target.

⚠️ Variable Lengths

Different questions have different token lengths. Using batch_size=1 sidesteps the need for padding — a simple but working solution for small datasets.

⚠️ Vocab = 324 Words

Entire vocabulary (questions + answers) = 324 unique tokens. This is both the embedding row count and the output dimension of the Linear layer.

⚠️ OHE vs Embeddings

Words are NOT one-hot encoded. They are mapped to learnable dense vectors (50-dim) via nn.Embedding. Much more efficient for large vocabularies.
