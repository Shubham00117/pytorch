04

Defining the Model — MySimpleNN

Full Class Definition
class MySimpleNN():

    def __init__(self, X):
        # Random weights, shape = [n_features, 1]
        self.weights = torch.rand(X.shape[1], 1,
                          dtype=torch.float64,
                          requires_grad=True)
        # Bias initialized to zero
        self.bias = torch.zeros(1,
                       dtype=torch.float64,
                       requires_grad=True)

    def forward(self, X):
        z = torch.matmul(X, self.weights) + self.bias
        y_pred = torch.sigmoid(z)
        return y_pred

    def loss_function(self, y_pred, y):
        # Clamp to avoid log(0)
        epsilon = 1e-7
        y_pred = torch.clamp(y_pred, epsilon, 1 - epsilon)
        # Binary Cross-Entropy Loss
        loss = -(y * torch.log(y_pred) +
                 (1 - y) * torch.log(1 - y_pred)).mean()
        return loss
Weights
  • torch.rand([30, 1])
  • dtype: float64
  • requires_grad = True
Forward Pass
  • Linear: matmul(X, W) + b
  • Activation: sigmoid(z)
  • Output range: (0, 1)
BCE Loss
  • Clamp predictions (ε = 1e-7)
  • Prevents log(0) = −∞
  • Returns scalar mean
💡 Why requires_grad=True? PyTorch uses this flag to track all operations on the tensor so it can automatically compute gradients during loss.backward().
05

Training Pipeline

Hyperparameters
learning_rate = 0.1
epochs = 25
Complete Training Loop
# Create model
model = MySimpleNN(X_train_tensor)

for epoch in range(epochs):

    # 1. Forward Pass
    y_pred = model.forward(X_train_tensor)

    # 2. Loss Calculation
    loss = model.loss_function(y_pred, y_train_tensor)
    print(f'Epoch: {epoch + 1}, Loss: {loss.item()}')

    # 3. Backward Pass (compute gradients)
    loss.backward()

    # 4. Parameter Update (no gradient tracking needed)
    with torch.no_grad():
        model.weights -= learning_rate * model.weights.grad
        model.bias    -= learning_rate * model.bias.grad

    # 5. Zero Gradients (prevent accumulation)
    model.weights.grad.zero_()
    model.bias.grad.zero_()
Step-by-Step Explanation
  • Forward Pass — Compute predictions using current W & b
  • Loss Calc — Measure how wrong predictions are (BCE)
  • Backward Pass — loss.backward() computes ∂L/∂W and ∂L/∂b via Autograd
  • Param Update — Gradient descent inside torch.no_grad() to skip tracking
  • Zero Grads — grad.zero_() prevents gradient accumulation across epochs
Update Equations
W_new = W_old − lr · (∂L/∂W)
b_new = b_old − lr · (∂L/∂b)

lr = learning rate = 0.1 | gradients computed by PyTorch Autograd

07

Model Evaluation

Evaluation Code
with torch.no_grad():
    # Run forward pass on test data
    y_pred = model.forward(X_test_tensor)

    # Threshold at 0.5 → convert to 0 or 1
    y_pred = (y_pred > 0.5).float()

    # Compare with ground truth
    accuracy = (y_pred == y_test_tensor).float().mean()
    print(f'Accuracy: {accuracy.item()}')
Final Accuracy (25 epochs, lr=0.1)
51.47%
⚠️ Low accuracy is expected — the goal of this video is to understand the training pipeline structure, not optimize performance.
More epochs, better lr tuning, and a proper architecture would improve this significantly.
Why Only ~51%?
  • Only 25 epochs — model hasn't converged
  • Single neuron (no hidden layers) — limited capacity
  • No lr scheduling or optimization (e.g., Adam)
  • Manual gradient updates (not using torch.optim)
  • This is intentional — foundational learning exercise
