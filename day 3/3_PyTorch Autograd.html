PyTorch Autograd

Automatic Differentiation · Computational Graphs · Gradient Computation

01 — What is Autograd
Definition

Autograd is a core component of PyTorch that provides automatic differentiation for tensor operations. It enables gradient computation essential for training ML models via gradient descent.

💡 Instead of manually deriving and coding derivatives, Autograd tracks all operations on tensors and automatically computes gradients via the chain rule.
Key Attribute

Set requires_grad=True to tell PyTorch to track operations on a tensor.

x = torch.tensor(3.0, requires_grad=True)
# tensor(3., requires_grad=True)

y = x**2
# tensor(9., grad_fn=<PowBackward0>)
grad_fn

Every computed tensor gets a grad_fn — a reference to the function that created it, used during backward pass.

Operation grad_fn
power (x**2) PowBackward0
sin (torch.sin) SinBackward0
addition (+) AddBackward0
sigmoid SigmoidBackward0
mean MeanBackward0
negation NegBackward0
02 — Why Autograd Matters
The Problem with Manual Derivatives

For simple functions, derivatives are easy. But nested functions get complex fast:

y = x² → dy/dx = 2x ✓ simple
y = x², z = sin(y), u = eᶻ → du/dx = (du/dz)·(dz/dy)·(dy/dx) ← complex chain rule

Writing code for every nested derivative manually is error-prone and impractical for deep neural networks. Autograd automates this entirely.

03 — Computational Graph (DAG)
How Autograd Tracks Operations

Autograd maintains a Directed Acyclic Graph (DAG) of Function objects.

x (leaf)
→
sq (x²)
→
y = 9
←← backward
dy/dx = 6
⏩ Forward pass: Runs the operation AND maintains gradient function in the DAG.
⏪ Backward pass: .backward() traverses DAG, computes gradients via chain rule, accumulates in .grad.
🔁 DAGs are dynamic in PyTorch — graph is recreated from scratch after each .backward() call. Allows control flow changes at every iteration.
04 — Examples
Example 1 — y = x²
x = torch.tensor(3.0, requires_grad=True)
y = x**2
y.backward()
print(x.grad)
# tensor(6.)

Manual check: dy/dx = 2x = 2(3) = 6 ✓

y = x²
dy/dx = 2x
at x=3: dy/dx = 6

Backwards: gradient of (2x) boxed, at x=3 → 2(3) = 6

Example 2 — Chain Rule: y = x², z = sin(y)
x = torch.tensor(4.0, requires_grad=True)
y = x ** 2          # PowBackward0
z = torch.sin(y)    # SinBackward0
z.backward()
print(x.grad)
# tensor(-7.6613)
dz/dx = (dz/dy)·(dy/dx)
dz/dy = cos(y) = cos(x²)
dy/dx = 2x
∴ dz/dx = 2x·cos(x²)

At x=4: 2(4)·cos(16) = -7.6613

Manual verify code: def dz_dx(x): return 2*x*math.cos(x**2) → dz_dx(2) = -2.6146, dz_dx(4) = -7.6613 ✓

05 — Neural Network (Example 3)
Training Process — 4 Steps
  • 1Forward pass — Compute output of the network given an input
  • 2Calculate loss — Quantify the error using a loss function
  • 3Backward pass — Compute gradients of the loss w.r.t. parameters
  • 4Update gradients — Adjust parameters using optimization (gradient descent)
Single Neuron — Forward Pass Formulas

1. LINEAR TRANSFORMATION

z = w·x + b

2. SIGMOID ACTIVATION

σ(z) = 1 / (1 + e⁻ᶻ)

3. BCE LOSS

L = −[y·ln(ŷ) + (1−y)·ln(1−ŷ)]
Backward Pass — Chain Rule Gradients
∂L/∂w = (∂L/∂ŷ)·(∂ŷ/∂z)·(∂z/∂w)
∂L/∂w = (ŷ − y) × x

where ∂z/∂w = x

∂L/∂b = (∂L/∂ŷ)·(∂ŷ/∂z)·(∂z/∂b)
∂L/∂b = (ŷ − y) × 1

where ∂z/∂b = 1

Intermediate: ∂L/∂ŷ = (ŷ−y)/[ŷ(1−ŷ)],   ∂ŷ/∂z = ŷ(1−ŷ)  → these cancel nicely!

06 — Autograd vs Manual: NN Example
Setup — x=6.7, y=0.0, w=1.0, b=0.0

MANUAL GRADIENT (no requires_grad) 

# Forward
z = w * x + b           # 6.7
y_pred = sigmoid(z)     # 0.9988
loss = bce(y_pred, y)   # 6.7012

# Backward (manual chain rule)
dloss_dy_pred = (y_pred - y) / (y_pred*(1-y_pred))
dy_pred_dz    = y_pred * (1 - y_pred)
dz_dw = x;  dz_db = 1

dL_dw = dloss_dy_pred * dy_pred_dz * dz_dw
dL_db = dloss_dy_pred * dy_pred_dz * dz_db

# dw = 6.6918,  db = 0.9988

AUTOGRAD (requires_grad=True)

w = torch.tensor(1.0, requires_grad=True)
b = torch.tensor(0.0, requires_grad=True)

# Forward (builds computation graph)
z      = w * x + b
# grad_fn=<AddBackward0>
y_pred = torch.sigmoid(z)
# grad_fn=<SigmoidBackward0>
loss   = bce(y_pred, y)
# grad_fn=<NegBackward0>

# One call does it all!
loss.backward()

print(w.grad)  # tensor(6.6918)
print(b.grad)  # tensor(0.9988)
✅ Manual == Autograd (Results Match)
dw = 6.6918
db = 0.9988

07 — Vector / Multivariate Gradients
Gradient w.r.t. Vector Input
x = torch.tensor([1.0, 2.0, 3.0], requires_grad=True)
y = (x**2).mean()
# tensor(4.6667, grad_fn=<MeanBackward0>)

y.backward()
print(x.grad)
# tensor([0.6667, 1.3333, 2.0000])
y = (x₁² + x₂² + x₃²) / 3
∂y/∂x₁ = 2x₁/3 = 2/3 ≈ 0.6667
∂y/∂x₂ = 2x₂/3 = 4/3 ≈ 1.3333
∂y/∂x₃ = 2x₃/3 = 6/3 = 2.0000

For multivariate functions, .backward() computes partial derivatives w.r.t. each input element simultaneously.

08 — Gradient Accumulation & Clearing
⚠️ Gradients Accumulate by Default!
x = torch.tensor(2.0, requires_grad=True)
y = x ** 2
y.backward()
print(x.grad)  # tensor(4.)   ← 2*2

# Call backward AGAIN (without clearing)
y = x ** 2
y.backward()
print(x.grad)  # tensor(8.)   ← 4+4 ACCUMULATED!
⚠️ PyTorch accumulates gradients — .grad adds up each time .backward() is called. In training loops you must zero gradients before each step! 
# Fix: zero before next backward
x.grad.zero_()
# tensor(0.)  ✓

# In training loops:
optimizer.zero_grad()  # same idea
09 — Disabling Gradient Tracking
3 Ways to Stop Gradient Tracking

Useful for inference, evaluation, or when you don't need gradients (saves memory & computation).

OPTION 1

requires_grad_(False) 
x.requires_grad_(False)
# tensor(2.)
# no requires_grad

# y.backward() → RuntimeError!

Permanently disables gradient tracking on the tensor in-place.

OPTION 2

x.detach() 
z = x.detach()
# tensor(2.) — no grad

# x still tracks grads
# z is detached copy

y = x ** 2
# still has grad_fn ✓

Returns a new tensor sharing data but detached from the graph.

OPTION 3

torch.no_grad() 
with torch.no_grad():
    y = x ** 2
    # no grad_fn built
    # no graph overhead

# Most common in inference!

Context manager — all ops inside skip graph building.

✅ Use torch.no_grad() during model evaluation / inference to speed up computation and reduce memory usage. Use detach() when you need a numpy-compatible tensor from a graph tensor.
10 — Quick Reference Summary
Concept Code Notes
Enable tracking tensor(..., requires_grad=True) Enables gradient computation for this tensor
Backward output.backward() Computes all gradients via chain rule
Read gradient x.grad Accumulated gradient stored here
Clear gradient x.grad.zero_() Must do before next backward in loops
Disable (perm) x.requires_grad_(False) In-place, tensor loses grad permanently
Detach z = x.detach() New tensor, same data, no graph
No grad ctx with torch.no_grad(): Best for inference — no graph built
