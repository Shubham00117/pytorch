03

CustomDataset Class

class CustomDataset(Dataset):

    def __init__(self, features, labels):
        self.features = torch.tensor(features, dtype=torch.float32)
        self.labels   = torch.tensor(labels,   dtype=torch.long)

    def __len__(self):
        return len(self.features)

    def __getitem__(self, index):
        return self.features[index], self.labels[index]

# Create dataset objects
train_dataset = CustomDataset(X_train, y_train)
test_dataset  = CustomDataset(X_test,  y_test)
3 Dunder Methods Required by PyTorch Dataset
  • __init__ — store features & labels as float32 and long tensors
  • __len__ — return total number of samples
  • __getitem__ — return one (feature, label) pair by index
⚠️
Features must be torch.float32; labels must be torch.long (int64) for CrossEntropyLoss to work correctly.

class MyNN(nn.Module):

    def __init__(self, num_features):
        super().__init__()
        self.model = nn.Sequential(
            nn.Linear(num_features, 128),
            nn.ReLU(),
            nn.Linear(128, 64),
            nn.ReLU(),
            nn.Linear(64, 10)   # 10 output logits (no Softmax)
        )

    def forward(self, x):
        return self.model(x)
💡
No Softmax at the end — nn.CrossEntropyLoss applies log-softmax internally. Adding Softmax manually would double it.

07

Training Loop

for epoch in range(epochs):
    total_epoch_loss = 0

    for batch_features, batch_labels in train_loader:

        # 1. Forward pass
        outputs = model(batch_features)

        # 2. Calculate loss
        loss = criterion(outputs, batch_labels)

        # 3. Back pass — zero grads first!
        optimizer.zero_grad()
        loss.backward()

        # 4. Update weights
        optimizer.step()

        total_epoch_loss += loss.item()

    avg_loss = total_epoch_loss / len(train_loader)
    print(f'Epoch: {epoch + 1} , Loss: {avg_loss}')
4-Step Inner Loop (per batch)
➡️
outputs = model(batch_features)
Forward pass — compute predictions
📉
loss = criterion(outputs, batch_labels)
Compute CrossEntropy loss
🔄
optimizer.zero_grad() → loss.backward()
Zero old gradients, then backpropagate
⬆️
optimizer.step()
Update all model weights via SGD
⚠️
optimizer.zero_grad() must be called before loss.backward() — PyTorch accumulates gradients by default. Skipping this causes incorrect weight updates.

Evaluation

# Switch to eval mode (disables dropout/batchnorm if used)
model.eval()

total   = 0
correct = 0

with torch.no_grad():    # disable gradient computation
    for batch_features, batch_labels in test_loader:

        outputs = model(batch_features)

        # Get predicted class (index of max logit)
        _, predicted = torch.max(outputs, 1)

        total   += batch_labels.shape[0]
        correct += (predicted == batch_labels).sum().item()

accuracy = correct / total
print(accuracy)
83.25%
Test Accuracy
🔍
torch.max(outputs, 1) returns (values, indices). We only need indices — the predicted class. _ discards the values. torch.no_grad() saves memory and compute since no backprop is needed.

Quick Reference

Key PyTorch Concepts Used
  • Dataset — abstract base class; implement __len__ + __getitem__
  • DataLoader — wraps Dataset; handles batching, shuffling, parallel loading
  • nn.Module — base for all neural network classes; implement forward()
  • nn.Sequential — chains layers linearly; no skip connections needed here
  • nn.CrossEntropyLoss — combines LogSoftmax + NLLLoss; expects raw logits
  • optim.SGD — Stochastic Gradient Descent optimizer
  • loss.backward() — computes gradients via autograd
  • optimizer.zero_grad() — clears accumulated gradients before each step
  • optimizer.step() — applies computed gradients to update weights
  • torch.no_grad() — context manager to skip gradient tracking during eval
  • model.eval() — sets model to evaluation mode (affects Dropout, BatchNorm)
  • torch.max(tensor, dim=1) — returns (max_values, argmax_indices) along class dim
