Problems with full-batch training: (1) Memory inefficient — loads all data at once   (2) Poor convergence — no stochastic gradient noise
Simple Solution — Manual Batching

Loop over data in chunks manually, without DataLoader.

# 320 samples / batch_size 32 = 10 batches per epoch
batch_size = 32
epochs = 25
n_samples = len(X_train_tensor)

for epoch in range(epochs):
    for start_idx in range(0, n_samples, batch_size):
        end_idx = start_idx + batch_size
        X_batch = X_train_tensor[start_idx:end_idx]
        y_batch = y_train_tensor[start_idx:end_idx]

        y_pred = model(X_batch)
        loss = loss_function(y_pred, y_batch.view(-1, 1))
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
    print(f"Epoch: {epoch+1}, Loss: {loss.item()}")
Problem: Manual, no shuffling, no parallelism → use Dataset + DataLoader instead.
Dataset & DataLoader — Core Concept
Key Idea: Dataset and DataLoader are core abstractions that decouple how you define your data from how you efficiently iterate over it in training loops.
Dataset Class (Blueprint)

Abstract class you subclass to define data. Must implement:

  • __init__ — how data is loaded
  • __len__ — total number of samples
  • __getitem__(index) — return (feature, label) at index
DataLoader Class (Iterator)

Wraps a Dataset and handles:

  • Batching into chunks of batch_size
  • Shuffling indices each epoch
  • Parallel loading via num_workers
  • Combining samples via collate_fn
DataLoader Control Flow (per epoch)
1
Sampler generates a shuffled list of all indices (if shuffle=True)
2
Indices are divided into chunks of batch_size — e.g. [4,6] [1,3] [7,8] [2,0] [9,5]
3
For each chunk, __getitem__ is called per index → fetches (feature, label) from Dataset
4
collate_fn stacks individual samples into a single batch tensor
5
Batch is returned to the training loop → batch1, batch2, …, batchN
6
Epoch ends when all indices are consumed. Next epoch reshuffles and repeats.
CustomDataset Implementation
from torch.utils.data import Dataset, DataLoader

class CustomDataset(Dataset):   # inherit from abstract Dataset

    def __init__(self, features, labels):
        self.features = features
        self.labels   = labels

    def __len__(self):
        return self.features.shape[0]   # total rows

    def __getitem__(self, index):
        return self.features[index], self.labels[index]

# Instantiate
dataset    = CustomDataset(X, y)
len(dataset)      # → 10
dataset[2]        # → (tensor([-2.8954, 1.9769]), tensor(0))

# Wrap in DataLoader
dataloader = DataLoader(dataset, batch_size=2, shuffle=True)

for batch_features, batch_labels in dataloader:
    print(batch_features)
    print(batch_labels)
DataLoader — Important Parameters
Parameter Default Description
dataset required Subclass of torch.utils.data.Dataset; must have __getitem__ & __len__
batch_size 1 Samples per batch. Larger = faster GPU training but more memory.
shuffle False Shuffles indices every epoch → prevents order-dependency
num_workers 0 Worker processes for parallel data loading. >0 leverages multiple CPU cores.
pin_memory False Copies tensors to pinned (page-locked) memory → faster GPU transfer on CUDA
drop_last False Drops final incomplete batch if n_samples % batch_size ≠ 0 (needed for BatchNorm)
collate_fn default stack Custom function to combine samples → useful for variable-length sequences (NLP)
sampler — Controls index-drawing strategy. Overrides shuffle.
Samplers
SequentialSampler

Draws indices in order 0,1,2,…N
Default when shuffle=False

RandomSampler

Draws indices randomly without replacement
Default when shuffle=True

Custom Samplers — useful for: imbalanced datasets (e.g. 1% vs 99% class split), time-series data where order matters, stratified sampling.
Parallelization with num_workers
Example: 10,000 samples · batch_size=32 · num_workers=4 → ~312 batches
1
Sampler generates shuffled indices → groups into 312 batches of 32. Queued for workers.
2
Parallel Loading: 4 workers each fetch 1 batch simultaneously — each calls __getitem__, applies transforms, runs collate_fn.
3
First batch returned: Whichever worker finishes first yields its batch to the training loop.
4
Model trains on batch 1 while workers 2/3/4 are already loading batches 2/3/4 — pipeline effect.
5
Continuous: Finished worker immediately grabs batch 5 from queue. At any moment, up to 4 batches being prepared.
7
End of epoch: All 312 batches consumed → DataLoader exhausted. If shuffle=True, reshuffles and repeats.
Data Transformations

__getitem__ is the transformation hook — apply transforms here before returning data.

Image Data
  • Resize
  • Grayscale / normalize
  • Data augmentation (flip, crop)
Text Data
  • Tokenization
  • Lemmatization
  • Stemming
collate_fn handles variable-length sequences (e.g., sentences tokenized to different lengths → pad to same length before stacking).

# Data prep
df = pd.read_csv('...')
df.drop(columns=['id', 'Unnamed: 32'], inplace=True)
X_train, X_test, y_train, y_test = train_test_split(
    df.iloc[:, 1:], df.iloc[:, 0], test_size=0.2)

scaler  = StandardScaler()
X_train = scaler.fit_transform(X_train)
X_test  = scaler.transform(X_test)

encoder = LabelEncoder()              # M→1, B→0
y_train = encoder.fit_transform(y_train)
y_test  = encoder.transform(y_test)

# To tensors (455 × 30)
X_train_tensor = torch.from_numpy(X_train.astype(np.float32))
y_train_tensor = torch.from_numpy(y_train.astype(np.float32))

# Dataset & DataLoader
train_dataset = CustomDataset(X_train_tensor, y_train_tensor)
test_dataset  = CustomDataset(X_test_tensor,  y_test_tensor)

train_loader = DataLoader(train_dataset, batch_size=32, shuffle=True)
test_loader  = DataLoader(test_dataset,  batch_size=32, shuffle=True)
Model 
class MySimpleNN(nn.Module):
    def __init__(self, num_features):
        super().__init__()
        self.linear  = nn.Linear(num_features, 1)
        self.sigmoid = nn.Sigmoid()

    def forward(self, features):
        out = self.linear(features)
        return self.sigmoid(out)

model          = MySimpleNN(X_train_tensor.shape[1])
optimizer      = torch.optim.SGD(model.parameters(), lr=0.1)
loss_function  = nn.BCELoss()
Training Loop 
for epoch in range(25):
    for batch_features, batch_labels in train_loader:
        y_pred = model(batch_features)
        loss   = loss_function(y_pred, batch_labels.view(-1, 1))
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
    print(f"Epoch: {epoch+1}, Loss: {loss.item()}")
Evaluation 
model.eval()
accuracy_list = []
with torch.no_grad():
    for batch_features, batch_labels in test_loader:
        y_pred       = model(batch_features)
        y_pred       = (y_pred > 0.8).float()    # threshold
        batch_acc    = (y_pred.view(-1) == batch_labels).float().mean().item()
        accuracy_list.append(batch_acc)

overall_accuracy = sum(accuracy_list) / len(accuracy_list)
print(f"Accuracy: {overall_accuracy:.4f}")
