Goal: reduce the train/test accuracy gap by adding a penalty term (Loss + Penalty) that discourages overly complex weight configurations.

⚡
Dropout
  • Applied to the hidden layers of the network.
  • Applied after the ReLU activation function.
  • Randomly turns off p% of neurons in the hidden layer during each forward pass.
  • Acts as a regularization technique — prevents co-adaptation of neurons.
  • Disabled during evaluation (inference mode) — all neurons are active.
PyTorch Implementation — Dropout only
class MyNN(nn.Module):
    def __init__(self, num_features):
        super().__init__()
        self.model = nn.Sequential(
            nn.Linear(num_features, 128),
            nn.ReLU(),
            nn.Dropout(p=0.3),         # 30% neurons dropped
            nn.Linear(128, 64),
            nn.ReLU(),
            nn.Dropout(p=0.3),
            nn.Linear(64, 10)
        )
    def forward(self, x):
        return self.model(x)
⚖️
Batch Normalization
  • Applied to hidden layers — not the output layer.
  • Goes after Linear, before Activation (Linear → BN → ReLU).
  • Normalizes activations using mini-batch mean & variance.
  • Introduces learnable γ (scale) and β (shift) parameters.
  • Reduces internal covariate shift → allows higher learning rates.
  • Regularization effect — noise from per-mini-batch statistics.
  • During evaluation: uses running mean & variance from training (not recomputed).
Full Architecture — Dropout + BatchNorm
Linear(784→128) → BatchNorm1d(128) → ReLU → Dropout(0.3) → Linear(128→64) → BatchNorm1d(64) → ReLU → Dropout(0.3) → Linear(64→10) 
class MyNN(nn.Module):
    def __init__(self, num_features):
        super().__init__()
        self.model = nn.Sequential(
            nn.Linear(num_features, 128),
            nn.BatchNorm1d(128),       # normalize after linear
            nn.ReLU(),
            nn.Dropout(p=0.3),
            nn.Linear(128, 64),
            nn.BatchNorm1d(64),        # normalize after linear
            nn.ReLU(),
            nn.Dropout(p=0.3),
            nn.Linear(64, 10)          # no BN on output layer
        )
    def forward(self, x):
        return self.model(x)
🔒
L2 Regularization (Weight Decay)
  • Applied to model weights — penalizes large values to encourage smaller, more generalizable weights.
  • Distributes learning across multiple parameters; avoids over-reliance on a few large weights.
  • No effect on bias terms — biases don't directly affect model complexity.
  • Active only during training — no effect during inference.
Two Ways to Apply
Lossreg = Lossoriginal + λ Σ wᵢ²
Via Loss Function — adds L2 penalty term
w ← w − η(∇Loss + λw)
Via Weight Decay — modifies gradient update directly
Hyperparameter
  • λ (lambda) — controls regularization strength. In PyTorch optimizer: weight_decay=λ. Larger λ → stronger penalty.
⚠ Weight decay in Adam ≠ L2 regularization exactly (use AdamW for decoupled weight decay)
