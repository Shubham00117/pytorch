📦 The torch.nn Module

torch.nn is a core library providing classes and functions to build neural networks efficiently. It abstracts away the complexity of creating and training networks by offering pre-built layers, loss functions, activation functions, and other utilities.

Key Components

① Modules / Layers

nn.Module — base class for all models.
Common: nn.Linear, nn.Conv2d, nn.LSTM

② Activation Functions

nn.ReLU, nn.Sigmoid, nn.Tanh — introduce non-linearities

③ Loss Functions

nn.BCELoss, nn.MSELoss, nn.CrossEntropyLoss

④ Container Modules

nn.Sequential — stack layers in order

⑤ Regularization

nn.Dropout, nn.BatchNorm2d — prevent overfitting

🏗️ Model Definitions

Simple Model (1 Linear Layer)

# input: 5 features → output: 1
import torch
import torch.nn as nn

class Model(nn.Module):

    def __init__(self, num_features):
        super().__init__()
        self.linear = nn.Linear(num_features, 1)
        self.sigmoid = nn.Sigmoid()

    def forward(self, features):
        out = self.linear(features)
        out = self.sigmoid(out)
        return out

# Create & run
features = torch.rand(10, 5)
model = Model(features.shape[1])
model(features)   # preferred over model.forward()
⚠️ Note: Always call model(x), NOT model.forward(x) directly — PyTorch hooks run only via __call__.

Improved Model v1 (2 Linear Layers)

class Model(nn.Module):

    def __init__(self, num_features):
        super().__init__()
        # Layer 1: features → 3
        self.linear1 = nn.Linear(num_features, 3)
        self.relu    = nn.ReLU()
        # Layer 2: 3 → 1
        self.linear2 = nn.Linear(3, 1)
        self.sigmoid = nn.Sigmoid()

    def forward(self, features):
        out = self.linear1(features)
        out = self.relu(out)
        out = self.linear2(out)
        out = self.sigmoid(out)
        return out

Improved Code v2 — nn.Sequential (Cleaner Syntax)

class Model(nn.Module):

    def __init__(self, num_features):
        super().__init__()
        self.network = nn.Sequential(
            nn.Linear(num_features, 3),
            nn.ReLU(),
            nn.Linear(3, 1),
            nn.Sigmoid()
        )

    def forward(self, features):
        out = self.network(features)
        return out
✅ Benefit: nn.Sequential eliminates manually wiring each layer in forward() — cleaner and less error-prone.

🔄 Training Loop — Old vs New Style

OLD Manual Weight Update

for epoch in range(epochs):

    # 1. Forward pass
    y_pred = model(X_train_tensor)

    # 2. Loss
    loss = model.loss_function(y_pred, y)

    # 3. Backward
    loss.backward()

    # 4. Manual update ← tedious
    with torch.no_grad():
        model.linear.weight -= lr * \
            model.linear.weight.grad
        model.linear.bias -= lr * \
            model.linear.bias.grad

    # 5. Zero gradients
    model.linear.weight.grad.zero_()
    model.linear.bias.grad.zero_()
Problem: Must manually update each parameter. Doesn't scale to deep networks.

NEW torch.optim Optimizer

# Setup (once)
loss_function = nn.BCELoss()
optimizer = torch.optim.SGD(
    model.parameters(), lr=learning_rate
)

for epoch in range(epochs):

    # 1. Forward pass
    y_pred = model(X_train_tensor)

    # 2. Loss (.view(-1,1) fixes shape)
    loss = loss_function(
        y_pred, y_train_tensor.view(-1, 1)
    )

    # 3. Clear gradients ← BEFORE backward
    optimizer.zero_grad()

    # 4. Backward pass
    loss.backward()

    # 5. Update all parameters
    optimizer.step()

    print(f'Epoch: {epoch+1}, Loss: {loss.item()}')
✅ Correct order: zero_grad() → backward() → optimizer.step() — zero grads BEFORE backward to avoid gradient accumulation from previous batch.

⚙️ The torch.optim Module

torch.optim provides a variety of optimization algorithms to update model parameters during training. It handles weight updates efficiently, including features like learning rate scheduling and weight decay (regularization).

model.parameters()

Returns an iterator over all trainable parameters (weights and biases) — instances of torch.nn.Parameter:
  • Weights — weight matrices of layers like nn.Linear
  • Biases — bias terms of layers

The optimizer uses these to compute gradients and update them during training.

Common Optimizers

Optimizer Usage
torch.optim.SGD Stochastic Gradient Descent
torch.optim.Adam Adaptive Moment Estimation
torch.optim.RMSprop Root Mean Square Propagation

optimizer = torch.optim.SGD(
